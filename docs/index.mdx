```lisp
(defn fizzbuzz (n)
  (dotimes (i n)
    (let k (+ i 1))
    (princ (cond ((is (mod k 15) 0) "FizzBuzz")
                 ((is (mod k 3)  0) "Fizz")
                 ((is (mod k 5)  0) "Buzz")
                 (else (number->string k))))
    (newline)))
(fizzbuzz 15)
```

## What it is

KEC Lisp is [Fe](https://github.com/rxi/fe) — rxi's tiny Lisp — plus a standard
library written in Lisp (`core/`) and a handful of portable C primitives
(`host/`). It ships as the interpreter, the library, a `kec` command-line tool,
and a test suite. The KN-86 firmware vendors it as a library and registers the
device primitives (graphics, sound, save state, missions, the CIPHER voice) on
top through the same FFI seam — none of that is here. This is just the language.

## A taste

```lisp
; bind with define / defn / let; mutate with set; compare with = / ==
(define greeting "deckline")
(str "hello, " greeting)             ; => "hello, deckline"  (str concatenates)

(map (fn (x) (* x x)) (range 1 6))   ; => (1 4 9 16 25)
(filter odd? (range 1 10))           ; => (1 3 5 7 9)
(fold-left + 0 (range 1 101))        ; => 5050
```

`nil` is the only false value and the empty list. Numbers are single-precision
floats. `=` and `==` are value equality; **assignment is `set`** (a deliberate
change from upstream Fe). See the [Language Reference](/kec-lisp/language/) for
the full story.
